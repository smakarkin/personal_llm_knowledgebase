# Immutable Raw Layer

## Зачем нужен raw layer
`raw/` — это минимальный «неизменяемый» слой источников. Его цель — отделить исходные материалы (статьи, книги, импорты и вложения) от рабочих заметок (`InBox`, `Zettelkasten`) и от generated LLM-слоёв (`10+`).

## Структура
- `raw/articles` — исходные материалы из статей, постов, публикаций.
- `raw/books` — заметки/фрагменты по книгам и главам.
- `raw/assets` — вложения: PDF, изображения, аудио, файлы данных.
- `raw/imports` — буфер для новых импортов до разборки.

## Принципы использования
1. **Immutable-by-default**: файлы в `raw/` не редактируются по смыслу после сохранения.
2. **Разрешены только безопасные правки**:
   - техническое переименование файла;
   - перенос между подпапками `raw/*`;
   - добавление служебного frontmatter (если нужно для поиска).
3. **Без массовых миграций**: старые файлы из `InBox`/`Zettelkasten` не переносятся автоматически.
4. **Рабочие выводы — вне raw**: обработка, обобщения и интерпретации делаются в `InBox`/`Zettelkasten`.
5. **LLM-скрипты не должны менять контент raw-источников**, кроме явно разрешённых служебных полей.

## Минимальный поток работы
1. Новый материал складывается в `raw/imports`.
2. После ручной разборки переносится в `raw/articles` или `raw/books`.
3. В `InBox` создаётся рабочая заметка с ссылкой на raw-источник.
4. Дальше запускается обычный LLM-пайплайн без миграции существующей базы.

## Индексация
Для быстрого обзора можно запускать:

```bash
python generate_raw_index.py
```

Скрипт создаёт `raw/RAW_INDEX.md` со списком markdown-файлов из `raw/articles`, `raw/books`, `raw/imports`.
