# Правила filing-back для 14_llm_traces

## Naming convention trace-файлов
- Формат: `Trace - YYYY-MM-DD HH-MM - <query-slug>.md`.
- Пример: `Trace - 2026-04-21 14-30 - retrieval-augmentation.md`.
- `<query-slug>` санитизируется: недопустимые символы Windows (`<>:"/\\|?*`) заменяются на `-`.

## Когда сохранять результат в trace
Сохраняем в `14_llm_traces`, если выполняется хотя бы одно:
1. Это одноразовый исследовательский запрос (mini-research), который может пригодиться как контекст позже.
2. Есть полезная связка между несколькими collections/concepts, но ещё рано фиксировать как устойчивое понятие.
3. Не хватает уверенности/источников для полноценного concept, но есть промежуточная гипотеза.
4. Нужна воспроизводимость: важно сохранить путь "запрос -> релевантные слои -> source notes -> синтез".

## Когда поднимать результат в concept
Поднимаем из trace в `12_llm_concepts`, если одновременно:
1. Гипотеза повторяется в нескольких запросах или явно устойчиво полезна.
2. Есть подтверждение через несколько source notes из разных коллекций/скоупов.
3. Формулировка уже достаточно стабильна для переиспользования в индексе/обзоре.
4. Можно дать понятное название concept из секции `# Понятие`.

## Минимальные требования к trace
Каждый trace-файл обязан содержать:
- исходный запрос;
- найденные collections/concepts;
- найденные source notes;
- объяснение релевантности;
- краткую synthesis/hypothesis *(если есть source notes и синтез действительно восстановлен)*.
